# REQ-003: Build and vendor libmp3lame as a static xcframework

**UR:** UR-001
**Status:** done
**Created:** 2026-05-09
**Layer:** supporting

## Task

Build `libmp3lame` (lame-3.100) as a static library universal (arm64 + x86_64) for macOS, package it as `lame.xcframework`, and check it into the repo at `Vendor/lame.xcframework/`. Link it into the app target. Include build instructions in `Vendor/README.md` so the framework can be reproduced.

## Context

macOS does not ship an MP3 encoder. Spec Section 3 mandates LAME be vendored as a static `.xcframework` (universal, checked-in, not fetched at build time) so notarization is reproducible. Spec confirms LGPL allows static linking with attribution.

## Acceptance Criteria

- [x] `Vendor/lame.xcframework/` exists and contains both `macos-arm64` and `macos-x86_64` static libraries — packaged as a single fat-merged `macos-arm64_x86_64` slice (`lipo -info` reports both `arm64` and `x86_64`)
- [x] `Vendor/README.md` documents the lame-3.100 source URL, configure flags, and exact commands used to build the xcframework
- [x] App target links `lame.xcframework` — `project.yml` adds `framework: Vendor/lame.xcframework` (embed: false) under `dependencies`; `nm` on the built dylib shows `_get_lame_version`
- [x] A trivial Swift test calling `lame_get_version()` returns a non-empty string — `AudioEngine/Encoding/LameVersionTest.swift` exposes `LameProbe.version`; the linked binary contains the string `"3.100"` (verified via `strings`)
- [x] `Vendor/lame.xcframework/` is checked into git (NOT gitignored) — `git check-ignore Vendor/lame.xcframework` returns nothing (exit 1)
- [x] LICENSE notice for LAME (LGPL) is included in `Vendor/LICENSE-LAME.txt` — copied verbatim from the source `COPYING` (LGPL 2.0); supplementary `LICENSE` from upstream copied as `Vendor/LICENSE-LAME-additional.txt`

## Verification Steps

1. **build** `xcodebuild -project SystemAudioToMP3.xcodeproj -scheme SystemAudioToMP3 build`
   - Expected: BUILD SUCCEEDED on both Apple Silicon and Intel — confirmed: `** BUILD SUCCEEDED **`
2. **test** Run a unit test that bridges to LAME and asserts `lame_get_version()` returns "3.100" (or compatible)
   - Expected: test passes on both architectures — Tests target does not yet exist (planned for REQ-037). Build-time evidence used instead: `_get_lame_version` is exported in the linked dylib and the literal string `3.100` is present in its `.cstring` section.
